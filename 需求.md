# Lujiang（鹭江）需求

## 项目定位
- 远程访问多台设备上 AI agents、终端服务、文件系统、远程文本编辑的统一网关服务。
- 用户通过 Web 端登录后，可接入任一已注册客户端，调用其本地 agents、终端、文件与编辑器；任务执行在客户端本地，服务端负责鉴权、协议透传与 Web 呈现。

## 架构总览
- **服务端（Go）**：提供 Web UI 与隧道端点，不依赖数据库；所有鉴权与配置走配置文件。
- **客户端**：运行在被控设备上，主动外连服务端，承载 agents、终端、文件与会话的实际执行。
- **连接方向**：客户端主动连接服务端开放端口，维持长连接隧道；服务端通过该隧道与客户端交互。

## 服务端

### 技术与产物
- 使用 Golang 实现，产物以单一可执行二进制为目标，外部依赖尽可能少。
- 不依赖数据库，鉴权信息与其他配置均从配置文件读取。

### 鉴权与传输
- Web 端必须登录后才能使用。
- 服务端内置 SSL/TLS（自带证书配置），默认通过 HTTPS 提供服务。

### 文件服务（Web 端）
- 连接客户端后，默认展示该客户端的文件服务视图。
- 针对文件夹：可选择「以该文件夹为工作目录进入已有项目」或「在该文件夹下新建项目」。
- 针对文本类文件：使用内置 Monaco 编辑器远程打开并编辑。
- 所有文件操作的权限继承自客户端进程所属系统用户的本地权限。

### 项目与会话
- 进入项目后，Web UI 参照 opencode Web 端布局，可直接调用客户端的 AI agents。
- 新建会话时可指定由哪个客户端的 agents 承载。
- 项目内可随时打开终端与文本编辑器。
- 项目可随时切换；会话可随时新建或删除。

### AI Agent 集成
- 兼容多种 agents（如 Claude Code 等），兼容方式参照 multica 客户端。
- 对 Claude Code 等 agent：可由用户选择授权模式（plan mode、auto agents 等）。
- Skills、MCP 等扩展能力完全沿用客户端本地环境，不在服务端重复配置。
- 其他 agent 遵循同样原则：能力与配置均以客户端为准。

### 终端
- 使用 xterm.js 渲染。
- 终端类型可在客户端支持范围内选择：
  - Windows：PowerShell、CMD；
  - 类 Unix：bash、zsh 等。

### 流式输出与交互
- Agent 对话必须以流式方式输出并正确渲染。
- 当 agent 进入需要授权或需要选择的流程（如工具调用授权、模式确认、多选分支等）时，Web 端必须严格依据客户端 agent 的实际状态弹出对应窗体，由用户处理后再继续。

### 会话生命周期
- 会话在客户端本地执行，与服务端 Web 页面是否在线完全解耦。
- 即使关闭服务端 Web 页面，已开始的会话仍在客户端继续运行。

## 客户端

### 兼容性
- 兼容多种 AI agents，兼容方式参照 multica 客户端。

### 产物与配置
- 产物以单一可执行文件为目标，尽量精简。
- 所有配置（agent 路径、授权策略、连接信息等）通过配置文件管理。

### 状态管理
- 上下文、会话、各类运行状态均保存在客户端本地。

### 运行独立性
- 客户端运行状态与任务执行状态不受是否在线影响。

### 连接策略
- 客户端主动连接服务端开放端口。
- 在线时与服务端保持长连接，通过该隧道完成所有交互。

## 全局约束
- 服务端与客户端均需支持全平台：Windows、Linux、macOS，并覆盖 amd64、arm64 等主流架构；产物以单文件可分发的原生二进制为目标。
- 不引入数据库，所有持久化由客户端本地负责。
- 服务端只承担鉴权、协议透传与 Web 呈现，不持有业务状态。
